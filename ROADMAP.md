# Jable Collect 产品路线图

> 更新时间：2026-03-30

---

## 优先级概览

| 优先级 | 功能 | 核心价值 |
|--------|------|----------|
| P0 | 增量同步 | 减少每次全量抓取的等待时间 |
| P0 | 筛选与排序增强 | 更快找到目标视频 |
| P0 | 同步历史面板 | 可观测，减少重复同步 |
| P1 | 标签 / 自定义分类 | 个人化整理，超越来源分类 |
| P1 | 批量操作 | 提高管理效率 |
| P1 | 数据修复工具 | 处理重复/损坏记录 |
| P2 | 详情侧栏 / 弹层 | 快速预览，不跳转页面 |
| P2 | 收藏视图优化 | 更舒适的浏览体验 |
| P2 | 导入导出增强 | 更好的数据迁移体验 |
| P3 | 每日随机 / 今日推荐 | 发现沉淀视频 |
| P3 | 智能整理 | 自动归类 / 去重建议 |
| P3 | 多站点扩展能力 | 接入更多视频站 |

---

## P0 — 核心体验（立即可做）

### 1. 增量同步

**问题**：每次同步都全量抓取所有页面，耗时长、有被封风险。

**方案**：
- 记录每条视频的 `syncedAt` 时间戳
- 再次同步时，遇到已存在且未变化的视频提前终止翻页
- UI 上区分「全量同步」和「增量同步」两个入口

**验收标准**：已收藏 200 条视频的用户，第二次同步时间 < 20 秒。

---

### 2. 筛选与排序增强

**问题**：目前仅支持来源筛选和简单排序，无法按演员、系列、关键词多维度过滤。

**方案**：
- 解析视频标题，提取演员名（通常在括号内）
- 增加演员筛选下拉框
- 增加按「收藏时间」「番号」排序选项
- 支持正则搜索

**验收标准**：用户可在 2 步以内定位某演员全部视频。

---

### 3. 同步历史面板

**问题**：用户不知道上次同步了多少条、是否成功。

**方案**：
- `background.js` 在每次 `syncFavorites` 后，追加一条记录到 `chrome.storage.local`（最多保留最近 20 条）
- Popup 页展示最近 5 条同步记录：时间、站点、数量、状态
- Options 页提供完整历史列表

**验收标准**：用户打开 Popup 即可看到最近一次同步结果。

---

## P1 — 管理效率（次季度）

### 4. 标签 / 自定义分类

**方案**：
- 在 IndexedDB schema 中增加 `tags: string[]` 字段
- Options 页每条视频右键菜单支持打标签
- 顶部增加标签筛选栏（多选）
- 标签数据随导出 JSON 一起携带

---

### 5. 批量操作

**方案**：
- 视频列表支持多选（复选框 + Shift 多选）
- 底部浮动操作栏：批量删除、批量打标签、批量导出
- 「全选当前页」「全选所有」快捷入口

---

### 6. 数据修复工具

**方案**：
- 检测 `url` 相同但字段不一致的重复记录
- 检测 `videoId` 为 null 的记录，尝试重新从 URL 提取
- 提供一键「修复」按钮，输出修复报告

---

## P2 — 体验打磨（下半年）

### 7. 详情侧栏 / 详情弹层

**方案**：
- 点击视频卡片展开侧栏，展示：封面大图、番号、标题、来源标记、标签、备注
- 支持在侧栏内直接打开原始页面
- 可选：嵌入预览视频播放

---

### 8. 收藏视图优化

**方案**：
- 支持「大图」「紧凑列表」「表格」三种视图切换
- 图片懒加载，避免一次性请求过多封面
- 封面加载失败时显示番号占位符

---

### 9. 导入导出增强

**方案**：
- 导出时支持选择字段（仅番号、含封面 URL、完整数据）
- 支持 CSV 导出，方便在 Excel 中查阅
- 导入时提供预览界面（将新增 N 条 / 更新 M 条）

---

## P3 — 探索功能（长期）

### 10. 每日随机 / 今日推荐

**方案**：
- Popup 页增加「随机看一条」按钮，从数据库随机取一条未近期访问的视频
- Options 页增加「每日精选」标签页，展示 5 条随机推荐

---

### 11. 智能整理

**方案**：
- 基于番号前缀（系列号）自动分组
- 对收藏超过 6 个月的视频生成「老片提醒」
- 检测已失效链接（404 检测）

---

### 12. 多站点扩展能力

**方案**：
- 抽象 `SiteScraper` 接口，让新站点只需实现 `getTotalPage()`、`parsePage()` 两个方法
- 在 Options 页提供站点管理面板
- 支持配置自定义站点的匹配规则（无需修改 manifest）

---

## 执行顺序建议

### 第一阶段（1-2 周）
1. 同步历史面板（改动小，用户价值立竿见影）
2. 增量同步（解决最痛的等待问题）

### 第二阶段（3-4 周）
3. 筛选与排序增强
4. 批量操作

### 第三阶段（5-8 周）
5. 标签系统
6. 详情侧栏
7. 数据修复工具

---

## 最小可行版本路线（MVP Subset）

如果只做 4 个功能，按此顺序：

1. **同步历史** — 可观测性，建立信任
2. **增量同步** — 效率，降低使用摩擦
3. **标签** — 个人化，核心差异化功能
4. **批量操作** — 管理效率闭环
